19 ## Mutation types
20 
21 The most basic type determining the behavior of a simulation is a mutation. Any __fwdpp__-based simulation must define a mutation type that publicy inherits from fwdpp::mutation_base. This base class contains several public data types:
22 
23 ~~~{.cpp}
24 //The mutation position
25 mutable double pos;
26 //The number of occurrences in the population
27 unsigned n;
28 //Does this mutation affect fitness/trait value?
29 bool neutral;
30 //This is used and updated internally by other functions w/in the library
31 bool checked;
32 ~~~
33 
34 This type is usable in a simulation, but not in an interesting simulation, as there is no selection coefficient, etc. In order to add things to a mutation type, you extend fwdpp::mutation_base via public inheritance. You may find an example of that in the next tutorial (@ref md_md_policies) and in the following types provided by the library:
35 
36 * fwdpp::mutation
37 * fwdpp::popgenmut
38 * fwdpp::generalmut
39 * fwdpp::generalmut_vec
40 
41 ### Notes
42 
43 * Remember to _properly_ initialize the base class from your derived mutation classes!
44 * All fitness models provided by the library require a mutation type to contain a double called \f$s\f$ representing the selection coefficient (or effect size). For example, see fwdpp::site_dependent_genetic_value. Hoewever, if you write your own fitness models, then your mutations can contain whatever they want.

46 ## Mutation containers
47 
48 Mutations must be stored in a container supporting random access via operator[]. Typically, this will be std::vector.
49 For example:
50 
51 ~~~{.cpp}
52 #include <vector>
53 #include <fwdpp/diploid.hh>
54 
55 //Create a typedef for a container of mutations:
56 using mcont_t = std::vector<fwdpp::mutation>;
57 ~~~

59 ## Gamete types
60 
61 The basic gamete type is fwdpp::gamete_base, which is a template type. A gamete contains two std::vector<std::size_t>
62 called _mutations_ and _smutations_. The values contained in these vectors correspond to the mutations present in the
63 gamete, and are used as indexes into the mutation container described above. The containers _mutations_ and
64 _smutations_ contain indexes (or "keys") to neutral and non-neutral mutations, respectively.
65 
66 A gamete also contains an unsigned integer, _n_, representing how many times the gamete is present in the simulation.
67 
68 __NOTE:__ __fwdpp__ makes no attempt to collect identical gametes together. In other words, imagine a recombination
69 event results in a gamete identical to one that already exists. The library will _not_ check for this, because the
70 comparison is relatively slow. Thus, the _n_ value will tend towards 1 over time, particularly for simulations with
71 large mutation/recombination rates.
72 
73 That's basically it for gametes.

80 ## Diploids
81 
82 In fwdpp, the simplest diploid is std::pair<std::size_t,std::size_t>. These two integers are indexes/keys referring to
83 the diploid's two gametes.
84 
85 The information represented in a diploid may be augmented via custom diploid types. See @ref md_md_customdip for
86 details.
87 
88 A simulation of a single deme makes use of a vector of diploids:
89 
90 ~~~{.cpp}
91 std::vector<std:pair<std::size_t,std::size_t> > diploids;
92 ~~~
93 
94 A simulation of multiple demes makes use of a vector of vector of diploids:
95 
96 ~~~{.cpp}
97 using diploid_t = std::pair<std::size_t,std::size_t>;
98 using dipvector_t = std::vector<diploid_t>;
99 std::vector<dipvector_t> diploids;
100 ~~~

102 ## Accessing the data in a population
103 
104 With the above definitions, we are ready to access all of the information in a population. Here's a quick run-through:
105 
106 ~~~{.cpp}
107 #include <fwdpp/diploid.hh>
108 #include <fwdpp/sugar/popgenmut.hpp>
109 #include <iostream>
110 
111 using mtype = fwdpp::popgenmut;
112 using mcont_t = std::vector<mtype>;
113 using gamete_t = fwdpp::gamete;
114 using gcont_t = std::vector<gamete_t>;
115 using diploid_t = std::pair<std::size_t,std::size_t>;
116 using dipcont_t = std::vector<diploid_t>;
117 
118 mcont_t mutations;
119 gcont_t gametes;
120 dipcont_t diploids;
121 
122 //Run a simulation, etc., now...
123 
124 //Go over all diploids, printing out the neutral and selected mutation
125 //positions of each variant in each diploid.
126 void print_mutations(const gamete_t::mutation_container & mc, const mcont_t & mutations)
127 {
128  for(const auto & key : mc) {
129  std::cout << mutations[key].pos << ' ';
130  }
131  std::cout << '\n';
132 }
133 
134 for(const auto & dip : diploids)
135 {
136  //neutral mutations, first gamete
137  print_mutations(gametes[dip.first].mutations,mutations);
138  //selected mutations, first gamete
139  print_mutations(gametes[dip.first].smutations,mutations);
140  //Repeat process for second gamete
141  print_mutations(gametes[dip.second].mutations,mutations);
142  print_mutations(gametes[dip.second].smutations,mutations);
143 }
144 
145 ~~~
