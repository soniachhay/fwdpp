10 /*! \namespace fwdpp::debug Debugging routines
11  * In general, the library does not throw exceptions
12  * during the execution of simulation-related algorithms.
13  * Rather, exception handling is limited to sanity-checking
14  * parameters inputs, etc., for object construction and other routines.
15  *
16  * However, it is sometimes useful to perform expensive checks on the
17  * data integrity of a simulated population. The routines in
18  * fwdpp::debug allow you to do that without affecting the
19  * performance of code compiled in "release" mode.
20  *
21  * Following the tradition of the C language, compiling with
22  * -DNDEBUG signals that a release-mode build is desired,
23  * meaning that expensive runtime checks are disables.
24  *
25  * If that symbol is not defined, then various runtime checks
26  * are enabled, and any failed checks with throw std::runtime_error.
27  * The message field of the error will begin with "FWDPP DEBUG:".
28  *
29  * Functions in namespace fwdpp::debug will be optimized out when
30  * code is compiled with -DNDEBUG, as they evaluate to empty functions.
31  */
